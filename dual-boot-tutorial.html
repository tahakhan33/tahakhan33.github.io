Windows/Linux Dual Boot Guide

Updated: July 2026 15 Min Read System Configuration

Configuring a workstation for computational physics often requires the best of both worlds. Dual booting allows you to retain Windows for specialized mechanical design software while harnessing a high-performance Linux environment (such as Ubuntu, Linux Mint, or Arch Linux) for compiling simulation codes and running environment management Bash scripts.

Step 1: Prepare the Windows Partition

Before installing Linux, you must free up unallocated space on your primary drive. Boot into Windows and open the Disk Management utility.

  • Right-click your main Windows partition (usually C:) and select Shrink Volume.
  • Allocate at least 50-100 GB of space for your Linux environment, depending on how many data libraries you plan to process.
  • Leave the newly freed space as "Unallocated."

Step 2: Create a Bootable USB

Download the ISO file for your preferred Linux distribution. Use a utility to flash this ISO onto a USB drive (8GB or larger).

# Commonly used flashing tools:
# Windows: Rufus or balenaEtcher
# Linux: dd command (if creating from another Linux machine)

Step 3: Adjust BIOS/UEFI Settings

To allow the Linux installer to run properly alongside Windows, restart your computer and enter the BIOS/UEFI settings (usually by pressing F2, F12, or DEL during startup).

  • Disable Secure Boot.
  • Disable Fast Boot (or Fast Startup within Windows settings).
  • Ensure the Boot Mode is set to UEFI (not Legacy/CSM) to match modern Windows installations.

Bootloader Tip: Installing Linux in UEFI mode ensures the GRUB bootloader will automatically detect your Windows Boot Manager, giving you a clean menu to choose your OS every time you power on the system.

Step 4: Install Linux

Boot from the USB drive and launch the Linux installer. When you reach the installation type or partitioning section:

  • Select the option to Install alongside Windows Boot Manager (this is automated in Ubuntu/Mint installers).
  • If using manual partitioning (common for Arch or Kali setups), create an ext4 or btrfs root partition (/) in the unallocated space, and ensure the bootloader targets the existing EFI System Partition.

Once the installation finishes, restart your machine, remove the USB drive, and use the GRUB menu to select your newly configured scientific computing environment.
