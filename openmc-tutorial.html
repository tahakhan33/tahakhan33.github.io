How to Install OpenMC on Linux

Updated: July 2026 15 Min Read Computational Physics

This guide provides a comprehensive, step-by-step methodology for building and installing the OpenMC Monte Carlo particle transport code from source on a Linux environment (Ubuntu/Debian based). It also covers setting up the necessary environmental variables.

Note: This tutorial assumes you have a working Linux terminal and basic sudo privileges.

Step 1: Install Prerequisites

Before compiling OpenMC, you must ensure your system has the correct compilers and build tools. Open your terminal and update your package list, then install git, cmake, and the build essentials.

sudo apt update
sudo apt upgrade -y
sudo apt install git cmake build-essential g++ libhdf5-dev

Step 2: Clone the OpenMC Repository

We will pull the latest source code directly from the official MIT/OpenMC GitHub repository. Navigate to the directory where you want to install it (e.g., your home folder or an /opt directory) and run:

git clone https://github.com/openmc-dev/openmc.git
cd openmc

Step 3: Build and Compile using CMake

OpenMC utilizes CMake to manage the build process. We will create a dedicated build directory to keep the source tree clean.

mkdir build
cd build
cmake ..
make -j4
sudo make install

The -j4 flag tells the compiler to use 4 CPU cores to speed up the build process. You can adjust this number based on your hardware capabilities.

Step 4: Configure Environment Variables

Once installed, Linux needs to know where to find the OpenMC executable. You should add the installation path to your .bashrc file.

echo 'export PATH=/usr/local/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

Step 5: Verify the Installation

To ensure OpenMC is correctly installed and accessible, type the following command in your terminal:

openmc --version

If the installation was successful, the terminal will output the current version of the OpenMC code.
