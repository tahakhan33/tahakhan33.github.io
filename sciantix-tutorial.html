Compiling & Running SCIANTIX

Updated: July 2026 10 Min Read Fission Gas Behavior

This tutorial outlines the procedure for downloading, compiling, and executing SCIANTIX, an open-source 0D code designed for the simulation of fission gas behavior in nuclear fuel.

Step 1: Install Prerequisites

SCIANTIX is written in C++ and uses CMake for its build system. Ensure that you have a modern C++ compiler and CMake installed on your Linux environment:

sudo apt update
sudo apt install git cmake build-essential g++

Step 2: Clone the Repository

Retrieve the latest source code from the official SCIANTIX GitLab repository. Navigate to your dedicated simulation workspace before running:

git clone https://gitlab.com/sciantix/sciantix.git
cd sciantix

Step 3: Build using CMake

Create an out-of-source build directory to keep the repository clean, then invoke CMake and Make to compile the executable.

mkdir build
cd build
cmake ..
make

Workflow Tip: After the build process completes, the compiled executable (usually named sciantix.x) will be located in your build directory. You can add this directory to your system's PATH variable for easier access.

Step 4: Running a Test Case

To execute the code, you need to provide the appropriate input files (e.g., input_settings.txt) in your working directory. You can test the installation by navigating to one of the provided regression test folders and running the executable:

./sciantix.x

If successful, SCIANTIX will generate output files containing the calculated fission gas release and bubble distribution data for post-processing.
